# Q&A: Спецификация и логика проекта

Этот документ содержит открытые вопросы по функционалу и архитектуре, которые требуют уточнения.

## Открытые вопросы (Требуют решения)

### 1. Юридическая ответственность и дисклеймеры
*   **Вопрос**: Как мы минимизируем юридические риски при выдаче "профессиональных" советов? 
*   **Варианты**:
    *   Обязательный чекбокс при регистрации с полным отказом от ответственности.
    *   Автоматическая блокировка советов, если ИИ видит ключевые слова (кровь, воспаление, опухоль).
    *   Постоянный баннер: "Не является медицинской рекомендацией".

### 2. Логика "Программы ухода" (Routine)
*   **Вопрос**: Насколько интерактивным должен быть план ухода?
*   **Варианты**:
    *   **Статичный**: ИИ выдает текст "Утром — пенка, вечером — крем".
    *   **Интерактивный (Трекер)**: Пользователь отмечает выполненные шаги каждый день. Это позволяет строить графики корреляции (например, "пропуск умывания -> ухудшение состояния через 2 дня").

### 3. Источник данных о продуктах
*   **Вопрос**: Откуда берутся конкретные названия средств?
*   **Варианты**:
    *   **Генерация ИИ**: ИИ предлагает что-то из своей базы знаний (риск галлюцинаций и неактуальных цен).
    *   **Собственный каталог**: Мы создаем таблицу в Supabase с проверенными средствами.
    *   **Партнерский API**: Интеграция с маркетплейсами для актуальных ссылок.

## Закрытые вопросы (Решения приняты)

*   **ИИ Провайдер**: Полный переход на **Gemini 3 Flash (OAuth)**.
*   **Безопасность**: Используем встроенные механизмы **Supabase (RLS, Auth)**.
*   **Платформа**: Только **Веб-приложение** (PWA и Cam-access отложены).
*   **Дизайн**: Текущий "Wow"-дизайн в стиле Glassmorphism устраивает.
*   **Лимиты**: Текущие лимиты Antigravity достаточны для этапа разработки.
*   **Стейт-менеджмент**: Ожидается решение по замене Redux Toolkit на более легкое (Zustand + Query).
